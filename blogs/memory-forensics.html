Introduction to Memory Forensics

Sep 28, 2024 8 min read

Traditional disk forensics focuses on recovering files from a hard drive. But what happens when malware never touches the disk? Enter Memory Forensics.

What is Memory Forensics?

Memory forensics is the analysis of volatile data in a computer's memory dump (RAM). It is crucial for identifying fileless malware, rootkits, encryption keys, injected code, and active network connections that exist solely in the computer's memory while powered on.

The Volatility Framework

The Volatility Framework is the industry standard for memory analysis. It is an open-source collection of tools implemented in Python under the GNU General Public License.

Key Artifacts Found in RAM

  • Running Processes: Not just what is running, but the parent-child relationships between processes, which helps spot anomalies (e.g., cmd.exe spawned by spoolsv.exe).
  • Network Connections: Active TCP/UDP connections and open ports, providing immediate indicators of command and control (C2) communication.
  • Injected Code: Identifying VAD (Virtual Address Descriptor) nodes with PAGE_EXECUTE_READWRITE permissions which often point to process hollowing or code injection.
  • Credentials: Passwords in clear text or password hashes that can be cracked offline.

Memory forensics requires a steep learning curve but is an indispensable skill for modern Incident Responders. As attackers continue to develop stealthy, fileless techniques, RAM analysis remains our most powerful magnifying glass.
